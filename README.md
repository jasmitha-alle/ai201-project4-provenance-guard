Provenance Guard

demo: https://drive.google.com/file/d/118wUV0F-hEbdrQWtgF7xCOdp1-ue38Lh/view?usp=drive_link

A backend API that classifies whether submitted creative content was written by a human or generated by AI, returns a confidence score, surfaces a transparency label, and handles creator appeals.


Setup

    python3 -m venv .venv
    source .venv/bin/activate
    pip install -r requirements.txt
    # add GROQ_API_KEY to .env
    PORT=5001 python app.py


Architecture

A submission flows through two signals in parallel, into a confidence scorer, then a label builder, and is written to the audit log before returning the response.

    POST /submit --> Signal 1 (Groq LLM) --> Confidence Scorer --> Label Builder --> Audit Log --> Response
                 --> Signal 2 (Heuristics) /

    POST /appeal --> Validate --> Status Update --> Audit Log --> Response


Detection Signals

Signal 1 is a Groq LLM classifier (weight 60%). It sends text to llama-3.3-70b-versatile and gets back P(AI-generated) as a float 0–1. It captures holistic patterns: hedging language, uniform register, lack of personal voice. Blind spot: formal human writing triggers false positives.

Signal 2 is a stylometric heuristic (weight 40%) combining three sub-measures — burstiness (CV of sentence lengths), type-token ratio (vocabulary diversity), and punctuation density. AI text is statistically flatter than human text across all three. Blind spot: texts under 40 words and intentionally repetitive poetry both defeat these signals.

These two signals are genuinely independent — one is semantic, one is structural — which makes the combination more informative than either alone.


Confidence Scoring

Confidence is separate from the combined AI score. It measures how certain the system is, penalising signal disagreement.

    combined     = (llm × 0.60) + (heuristic × 0.40)
    base         = |combined - 0.5| × 2
    disagreement = |llm - heuristic|
    penalty      = max(0, (disagreement - 0.15) / 0.85)
    confidence   = base × (1 - penalty × 0.5)

Thresholds: confidence >= 0.80 and combined >= 0.50 → likely_ai. Confidence >= 0.75 and combined < 0.50 → likely_human. Everything else → uncertain. The AI threshold is higher because a false positive is the worse error.

Two example submissions:

High confidence (0.80) — clearly AI text, both signals agree:
    content_id: bd40389a, attribution: likely_ai, confidence: 0.8, llm_score: 0.8

Lower confidence (0.369) — same text after heuristics added; LLM=0.80, heuristics=0.542, signals disagree, penalty applied:
    content_id: 4796e6e6, attribution: uncertain, confidence: 0.369, llm_score: 0.8, heuristic_score: 0.542


Transparency Labels

Likely AI-Generated (confidence >= 0.80, attribution = likely_ai):
"⚠️ Likely AI-Generated — Our system is [X]% confident this content was produced by an AI writing tool. It has been held for editorial review. If you wrote this yourself, see 'Appeal this decision' below."

Likely Human-Written (confidence >= 0.75, attribution = likely_human):
"✅ Likely Human-Written — Our system is [X]% confident this content was written by a person. No AI concerns detected. Cleared for publication."

Uncertain (all other cases):
"❓ Origin Uncertain — Our system could not confidently determine authorship ([X]% confidence). Queued for human review. If you wrote this yourself, see 'Appeal this decision' below."


Rate Limiting

10 requests per minute per IP on POST /submit. A creator submitting their own work will never hit this. A script flooding the system will. The appeal and log endpoints are not rate-limited — they don't call external APIs.

Audit Log

Every submission and appeal is logged to SQLite. Retrieve with GET /log. Sample entries:

    {"content_id": "4796e6e6", "attribution": "uncertain", "confidence": 0.369, "llm_score": 0.8, "status": "under_review", "extra": {"appeal_reasoning": "I wrote this myself. I am a non-native English speaker and my writing style may appear more formal than typical."}}
    {"content_id": "9fc575f1", "attribution": "uncertain", "confidence": 0.619, "llm_score": 0.2, "status": "classified", "extra": {"heuristic_score": 0.176}}
    {"content_id": "bd40389a", "attribution": "likely_ai", "confidence": 0.8, "llm_score": 0.8, "status": "classified"}


Known Limitations

Short text: under 40 words, TTR and burstiness fall back to 0.5 and almost all inputs return uncertain regardless of origin. The system should flag these as unanalyzable rather than returning a low-confidence result.

Formal human writing: academic prose and non-native English writing share surface features with AI output. Both signals fire toward AI on this content and a high-confidence false positive is possible. The 0.80 threshold reduces this but does not eliminate it.


Spec Reflection

The confidence formula written in planning.md before implementation meant the code was a direct translation — when llm=0.90/heuristic=0.40 produced confidence 0.318, it was immediately clear that was correct behavior, not a bug.

One divergence: the spec expected likely_human to appear regularly in testing. In practice almost all test inputs landed as uncertain because texts were short and heuristics fell back to 0.5. The 0.75 threshold is calibrated for longer passages.
